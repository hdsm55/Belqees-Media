# 🚀 اختبار سريع للاتصال بقاعدة البيانات

## الطريقة الأسهل: استخدام API Endpoint

تم إنشاء endpoint خاص للاختبار!

### الخطوة 1: تأكد من أن الخادم يعمل

افتح Terminal جديد وشغل:

```bash
npm run dev
```

### الخطوة 2: افتح المتصفح

انتقل إلى:

```
http://localhost:3000/api/test-db
```

### الخطوة 3: شاهد النتيجة

**إذا نجح الاتصال:**
- سترى رسالة: `"الاتصال بقاعدة البيانات نجح!"`
- سترى قائمة الجداول
- سترى عدد المستخدمين
- سترى إذا كان عمود `supabaseUserId` موجود

**إذا فشل الاتصال:**
- سترى رسالة الخطأ
- سترى تفاصيل المشكلة

---

## طريقة بديلة: Prisma Studio

```bash
npm run db:studio
```

إذا فتح Prisma Studio بنجاح، يعني الاتصال يعمل.

---

## طريقة أخرى: Terminal

```bash
npm run db:check
```

---

## ✅ بعد الاختبار

إذا نجح الاتصال من API:
1. ✅ قاعدة البيانات متصلة بشكل صحيح
2. ✅ يمكنك الآن تشغيل: `npx prisma db push`
3. ✅ يمكنك إنشاء حساب جديد

إذا فشل:
- تحقق من `DATABASE_URL` في `.env.local`
- تأكد من استخدام **Session Pooler** من Supabase
- تأكد من أن Supabase Project نشط

---

**جرب الآن:**
1. شغل `npm run dev`
2. افتح `http://localhost:3000/api/test-db`
3. شاهد النتيجة! 🎉
