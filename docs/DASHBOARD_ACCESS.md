# دليل الوصول إلى لوحة التحكم

## كيفية الوصول إلى لوحة التحكم

### الخطوة 1: إنشاء حساب جديد (إذا لم يكن لديك حساب)

1. افتح المتصفح وانتقل إلى:
   ```
   http://localhost:3000/register
   ```
   أو
   ```
   http://localhost:3000/login
   ```

2. إذا لم يكن لديك حساب، اضغط على "سجل الآن" أو انتقل مباشرة إلى `/register`

3. املأ البيانات:
   - **الاسم** (اختياري)
   - **البريد الإلكتروني** (مطلوب)
   - **كلمة المرور** (6 أحرف على الأقل)

4. اضغط على "إنشاء حساب"

5. بعد إنشاء الحساب، سيتم توجيهك إلى صفحة تسجيل الدخول

### الخطوة 2: تسجيل الدخول

1. انتقل إلى:
   ```
   http://localhost:3000/login
   ```

2. أدخل:
   - **البريد الإلكتروني**
   - **كلمة المرور**

3. اضغط على "تسجيل الدخول"

4. بعد تسجيل الدخول بنجاح، سيتم توجيهك تلقائياً إلى:
   ```
   http://localhost:3000/dashboard
   ```

## الروابط المباشرة

### صفحات المصادقة
- **تسجيل الدخول**: `http://localhost:3000/login`
- **إنشاء حساب**: `http://localhost:3000/register`

### صفحات لوحة التحكم
- **الصفحة الرئيسية**: `http://localhost:3000/dashboard`
- **إدارة الصفحات**: `http://localhost:3000/dashboard/pages`
- **إدارة الخدمات**: `http://localhost:3000/dashboard/services`
- **إدارة الأعمال**: `http://localhost:3000/dashboard/portfolio`
- **إدارة الفعاليات**: `http://localhost:3000/dashboard/events`
- **إدارة المدونة**: `http://localhost:3000/dashboard/blog`

## الحماية والأمان

### التحقق التلقائي
- إذا حاولت الوصول إلى `/dashboard` بدون تسجيل الدخول، سيتم توجيهك تلقائياً إلى `/login`
- بعد تسجيل الدخول، يمكنك الوصول إلى جميع صفحات لوحة التحكم

### الأدوار والصلاحيات
- **VIEWER**: يمكنه فقط عرض البيانات
- **CONTRIBUTOR**: يمكنه إنشاء وتعديل المحتوى
- **EDITOR**: يمكنه إنشاء وتعديل وحذف المحتوى
- **ADMIN**: صلاحيات كاملة + إدارة المستخدمين

> **ملاحظة**: عند إنشاء حساب جديد، يتم تعيين الدور الافتراضي كـ `VIEWER`. لتغيير الدور، يجب أن يقوم مدير النظام بتعديله من قاعدة البيانات أو من لوحة إدارة Supabase.

## استكشاف الأخطاء

### مشكلة: لا يمكنني تسجيل الدخول
1. تأكد من أن Supabase يعمل بشكل صحيح
2. تحقق من ملف `.env.local` وإعدادات Supabase
3. تأكد من أن البريد الإلكتروني وكلمة المرور صحيحة
4. تحقق من أن المستخدم موجود في Supabase

### مشكلة: يتم توجيهي إلى `/login` دائماً
1. تحقق من أن Session يعمل بشكل صحيح
2. تأكد من إعدادات Supabase في `.env.local`
3. تحقق من Cookies في المتصفح

### مشكلة: لا أستطيع الوصول إلى صفحة معينة
1. تحقق من دورك في قاعدة البيانات
2. بعض الصفحات تتطلب أدوار محددة (مثل إدارة المستخدمين تتطلب ADMIN)

## إعداد Supabase

تأكد من أن ملف `.env.local` يحتوي على:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
```

## إنشاء مستخدم Admin يدوياً

إذا كنت تريد إنشاء مستخدم Admin، يمكنك:

1. إنشاء حساب عادي من `/register`
2. فتح Supabase Dashboard
3. البحث عن المستخدم في جدول `users`
4. تحديث `role` إلى `ADMIN` في قاعدة البيانات

أو استخدام Prisma Studio:
```bash
npx prisma studio
```

ثم تحديث `role` في جدول `User` إلى `ADMIN`.
