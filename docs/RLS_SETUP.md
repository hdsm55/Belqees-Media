# 🔒 إعداد Row Level Security (RLS) - Belqees Media

## 📋 ما هو RLS؟

Row Level Security (RLS) هو نظام أمان في PostgreSQL يسمح بالتحكم في الوصول إلى البيانات على مستوى الصفوف. هذا يعني أن كل مستخدم يمكنه رؤية/تعديل البيانات المسموح له بها فقط.

---

## 🚀 طريقة التفعيل

### الخطوة 1: فتح Supabase SQL Editor

1. اذهب إلى [Supabase Dashboard](https://app.supabase.com)
2. اختر مشروعك
3. اذهب إلى **SQL Editor** من القائمة الجانبية
4. اضغط على **New Query**

### الخطوة 2: تشغيل السكريبت

انسخ محتوى ملف `scripts/enable-rls.sql` والصقه في SQL Editor، ثم اضغط **Run**.

أو يمكنك تشغيله مباشرة من خلال MCP:

```bash
# سيتم تنفيذ السكريبت تلقائياً
```

---

## 📊 السياسات (Policies) المطبقة

### 1. Users (المستخدمون)
- ✅ المستخدمون يمكنهم رؤية بياناتهم فقط
- ✅ المستخدمون يمكنهم تحديث بياناتهم فقط

### 2. Pages, Services, Portfolio, Events, Blog Posts
- ✅ الجميع يمكنهم قراءة المحتوى المنشور (`published = true`)
- ✅ المستخدمون المصرح لهم يمكنهم إدارة المحتوى

### 3. Categories, Tags, Media, Translations
- ✅ الجميع يمكنهم القراءة
- ✅ المستخدمون المصرح لهم يمكنهم الإدارة

### 4. Contact Messages
- ✅ الجميع يمكنهم إرسال رسائل
- ✅ فقط المستخدمون المصرح لهم يمكنهم قراءة الرسائل

---

## ✅ التحقق من التفعيل

بعد تشغيل السكريبت، يمكنك التحقق:

1. اذهب إلى **Database** → **Tables**
2. اختر أي جدول
3. اذهب إلى **Policies**
4. يجب أن ترى السياسات المضافة

---

## 🔍 اختبار RLS

### اختبار 1: قراءة البيانات
```sql
-- يجب أن يعمل بدون مشاكل
SELECT * FROM pages WHERE published = true;
```

### اختبار 2: محاولة الوصول بدون مصادقة
```sql
-- يجب أن يفشل إذا لم تكن مسجل دخول
SELECT * FROM contact_messages;
```

---

## ⚠️ ملاحظات مهمة

1. **RLS يعمل فقط مع Supabase Auth**: يجب أن يكون المستخدم مسجل دخول
2. **Policies قابلة للتخصيص**: يمكنك تعديلها حسب احتياجاتك
3. **اختبار قبل الإنتاج**: تأكد من اختبار جميع السيناريوهات

---

## 🔧 تحديث Policies

إذا أردت تحديث Policies:

1. اذهب إلى **Database** → **Tables** → **Table Name** → **Policies**
2. اضغط على **Edit** أو **Delete**
3. أو استخدم SQL Editor لتعديلها

---

**جاهز!** 🚀

بعد تفعيل RLS، ستكون قاعدة البيانات أكثر أماناً.
